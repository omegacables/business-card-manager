# Rork Max プロンプト例（iOSアプリ化）

このリポジトリを Rork Max に参照させて、iOSネイティブアプリを生成するためのプロンプト集。
**上から順に、1プロンプトずつ送って段階的に育てる**のがコツ（一度に全部指示すると精度が落ちる）。

> 前提：現状のAPIはWebのCookieセッション認証のため、**iOSアプリから直接ログインして叩くことはまだできない**。
> そのため Step 1〜3 はモックデータでアプリを完成させ、Step 4（API接続）はバックエンド側に
> Bearerトークン認証を追加してから行う（追加作業は本体リポジトリ側のタスク）。

---

## Step 1: アプリの土台（最初に送るプロンプト）

```
日本のビジネスパーソン向けの名刺管理アプリ「名刺管理」を作ってください。
仕様の正は GitHub リポジトリ omegacables/business-card-manager の README.md にある
「データモデル」と「APIエンドポイント」です。これに合わせてください。

## 画面構成（タブバー: ホーム / 名刺 / 登録 / 設定）
1. ホーム: 統計カード（登録済み名刺数・今月の登録数・プラン）、最近登録した名刺5件
2. 名刺一覧: 検索バー（名前・会社名・メール）、頭文字アバター付きリスト
3. 名刺詳細: 基本情報・連絡先・住所・メモ、名刺画像、「やり取りの履歴」タイムライン
   （打ち合わせ/電話/メール/LINE/メモ/タスクの種類別アイコン付き）
4. 新規登録: カメラで名刺撮影 → OCR結果の確認・編集フォーム（今はモックでOK）
5. 設定: プロフィール、LINE連携状態、プラン表示、テーマ切替

## データ
README の business_cards / activities スキーマと同じ構造の Swift モデルを定義し、
日本語のリアルなモックデータ（名刺15件、各名刺に活動履歴2〜5件）で全画面を動かしてください。
API接続は後で行うので、データ層は Repository プロトコルで抽象化してモック実装を注入する設計にしてください。

## デザイン
- 白ベース＋藍色（ジャパンブルー #274C8D 系）のアクセント、ダークモード対応（ダークはニュートラルグレー）
- iOSネイティブの作法に従う（SF Symbols、Dynamic Type、標準ナビゲーション）
- 余白を広めに、文字は大きめに、長時間見ても疲れない落ち着いたトーン
- 数字は等幅で表示し「◯件」のように単位を付ける
```

## Step 2: 名刺スキャン（カメラ + OCR）

```
新規登録画面を実装してください。
1. カメラまたはフォトライブラリから名刺画像を取得（VisionKit の DataScanner や AVFoundation を使用）
2. まずは端末上の Vision フレームワーク（VNRecognizeTextRequest、日本語対応）でOCRし、
   氏名・会社名・役職・電話・メールを推定してフォームに自動入力
3. ユーザーが確認・修正して保存（モックRepositoryに追加）
将来はサーバーの POST /api/ocr に切り替えられるよう、OCR処理も protocol で抽象化してください。
```

## Step 3: AI追いメール・タイムライン強化

```
名刺詳細に「AI追いメール」ボタンを追加してください。
- タップするとシート表示: トーン切替（ビジネス丁寧 / カジュアル）、下書き本文、
  「コピー」「メールアプリで開く」（MFMailComposeViewController）ボタン
- 今はモックの下書き文面を返す FollowUpService プロトコルで実装
  （将来 POST /api/cards/{id}/followup に接続。リクエスト: { "tone": "formal" | "casual" }、レスポンス: { "draft": "..." }）
また、タイムラインに手動でメモを追加するシート（種類選択＋テキスト）も付けてください。
```

## Step 4: API接続（バックエンド対応後に送る）

```
モックRepositoryを実データに切り替えます。
ベースURL: https://business-card-manager-ruddy.vercel.app
認証: Authorization: Bearer <アクセストークン>（トークンはログイン画面で取得、Keychainに保存）
エンドポイントは README.md の一覧どおり:
- GET /api/cards（一覧・検索）、POST /api/cards（登録）
- PUT/DELETE /api/cards/{id}
- POST /api/cards/{id}/followup（AI追いメール）
- POST /api/ocr（multipart、10MB以下、JPEG/PNG/WebP）
- GET /api/auth/me（プロフィール）
エラーは日本語で親切に表示し、401のときは再ログインに誘導してください。
APIキーや秘密情報をコードに埋め込まないでください。
```

## Step 5: App Store 対応

```
App Store 審査向けの仕上げをしてください。
- Sign in with Apple を追加（サードパーティログインを載せるための必須要件・ガイドライン4.8）
- カメラ・フォトライブラリの利用目的文（Info.plist の日本語説明）
- プライバシーポリシーへのリンクを設定画面に追加
- オフライン時の表示、読み込み中のスケルトン表示
```

## 運用メモ

- **1プロンプト=1テーマ**で送り、生成結果を確認してから次へ進む
- 修正指示は「◯◯画面の△△を□□に変えて」のように画面名で具体的に
- Rork Max で生成したコードはエクスポートして `business-card-ios`（別リポジトリ）で管理する
- **やってはいけないこと**: Supabase の service role キーや LINE のチャネルシークレットをアプリに渡すこと。
  アプリが直接触ってよいのは上記の自社API（と将来のログインエンドポイント）だけ
